What is RESTful API ?

REST APIs, or RESTful APIs as they're commonly called, are one of the most prevalent types of API. REST or RESTful APIs have numerous advantages because they are designed to take advantage of existing protocols. While REST - or Representational State Transfer - can be utilized across practically any protocol, it is most commonly employed for web APIs. This means that when developing a REST API, developers do not need to install any additional software or libraries.

On Time Delivery API with Node js and Express

Using this REST API, you can create, read, update, and delete a parcel by its id, or get and delete all courses. To simplify the example, I used an array to store the parcel objects.

This API has seven methods:

  • Get all parcels
  • Get a user parcel by id
  • Add a new parcel
  • Update a parcel by id
  • Delete a parcel by id
  • User sign up or authorization by post method
  • User logs in also by post method

My Swagger API Documentaion

I used Swagger to document the example REST API. swagger.jsdoc is the document file. (YMAL is also a popular language used to write Swagger Document for REST API documentation).

The advantage of using Swagger to document REST API is that it provides a dynamic document with "Try it out" buttons. Developers can click the "Try it out" buttons to see how the API works.

Writing a Swagger API specification file to define an API before starting to code is a good workflow for creating REST API and its document.
